Key Design Decisions Shown

  • Before/after: Raw input text stays visible (amber highlight on mobile, in input field on desktop) while extracted fields appear below/above.
  • Populated fields only: Only Title and Due date appear (extracted with confidence). Priority, location, recurrence hidden behind "+ Add" links.
  • surface-extracted tint: The subtle warm background on extracted fields signals "AI filled this."
  • Feedback controls: Thumbs up/down inline with save — unobtrusive, not blocking.

Key Design Decisions Shown

Key Design Decisions Shown

  • Count always visible: "14 this week" in the header. "3 today · 14 this week" in the completed section. No tap needed to see progress.
  • Amber warmth: Completed items have the surface-completed background. Checkmarks use amber-500. Text uses amber-900. The warmth is earned.
  • Expandable list: Completed section expands with chevron. The count is the daily signal; the full list is the receipt.
  • Desktop variation: Shows completed items in a warm card — giving them their own visual territory rather than inline with active tasks.

The Five States of the Capture Loop

  • IDLE: Task list visible, input ready at bottom with placeholder suggestion.
  • TYPING: Keyboard up, input focused with amber ring, submit button activates.
  • EXTRACTING: Input text moves to "before" display. Breathing shimmer appears in form area (only if >800ms). Conditional — fast extractions skip this state.
  • EXTRACTED: Populated fields snap in (250ms). Raw input still visible as "before." Save button prominent. Feedback controls inline.
  • SAVED: Task appears in list (optimistic). Input cleared. Cursor ready. New task has brief extracted tint that fades. Loop restarts.
